# 录音文件转文本（ASR）工具开发文档

> [!success] 目标：开发一个将录音文件转成文本的工具。

整体方案：把音频文件上传到火山引擎 **TOS 对象存储**，得到一个公网可访问的音频 URL，然后调用火山引擎「录音文件识别 2.0（AUC BigModel）」接口进行识别，最后输出 `识别结果.txt`。

- 主程序：asr.py
- 官方参考：`auc_python_example`（字节火山引擎官方示例工程）
- 官方接口文档：https://www.volcengine.com/docs/6561/1354868?lang=zh

---

## 1. 总体流程（从音频到文本）

1. **准备音频文件**
   - 如格式不符合要求：可先使用 Convertio 或 ffmpeg 转换格式
   - 如音频太大：可先用 ffmpeg 切分成小文件
2. **上传到 TOS 对象存储**
   - 上传后得到一个“浏览器能直接打开”的公网 URL
3. **提交识别任务（submit）**
   - 在请求体里传入 `audio.url`（TOS 公网 URL）与 `audio.format`
4. **轮询查询结果（query）**
   - 根据响应头 `X-Api-Status-Code` 判断任务完成/处理中/失败
5. **保存识别文本**
   - 成功后写入 `识别结果.txt`

## 2. asr.py 代码说明与关键点

### 2.1 需要修改的配置

代码把需要改的配置集中在顶部（asr.py:5-10）：

- `X_API_KEY`：火山引擎新版控制台获取的 **X-Api-Key**（注意是 X-Api-Key，不是 AppID）
- `FILE_URL`：音频在 TOS 的 **公网 URL**（确保浏览器能直接打开）

```python
X_API_KEY = "你的 X-Api-Key"
FILE_URL = "https://xxx.tos-cn-beijing.volces.com/xxx.mp3"
```

### 2.2 接口地址与资源 ID

代码中固定配置（如需变更以官方文档为准）：

- 提交接口：`SUBMIT_URL`（asr.py:13)）
- 查询接口：`QUERY_URL`（[asr.py:14](asr.py#L14)）
- 资源 ID：`RESOURCE_ID = "volc.seedasr.auc"`（[asr.py:15-16](asr.py#L15-L16)）

### 2.3 提交任务（submit_task）

```python
def submit_task():
    task_id = str(uuid.uuid4())
    # 🛑 核心修正：新版控制台只用X-Api-Key！
    headers = {
        "X-Api-Key": X_API_KEY,  # 新版请求头字段
        "X-Api-Resource-Id": RESOURCE_ID,
        "X-Api-Request-Id": task_id,
        "X-Api-Sequence": "-1",
        "Content-Type": "application/json"
    }
    # 最简请求体
    request_body = {
        "user": {"uid": "asr_user"},
        "audio": {
            "url": FILE_URL,
            "format": "mp3"
        },
        "request": {
            "model_name": "bigmodel",
            "enable_punc": True,
            "enable_itn": True
        }
    }
    print(f"提交任务，任务ID：{task_id}")
    try:
        response = requests.post(
            SUBMIT_URL,
            data=json.dumps(request_body),
            headers=headers,
            timeout=60
        )
        print(f"响应状态码：{response.status_code}")
        print(f"响应头：{dict(response.headers)}")
        print(f"响应内容：{response.text}")
        response.raise_for_status()
        status_code = response.headers.get("X-Api-Status-Code", "")
        if status_code == "20000000":
            print(f"✅ 任务提交成功！")
            x_tt_logid = response.headers.get("X-Tt-Logid", "")
            return task_id, x_tt_logid
        else:
            print(f"❌ 任务提交失败")
            exit(1)
    except Exception as e:
        print(f"❌ 提交异常：{str(e)}")
        exit(1)
```

关键实现：
- 生成 `task_id = uuid.uuid4()`，作为本次任务的唯一标识
- 请求头使用新版字段（核心是 `X-Api-Key`）：
  - `X-Api-Key`
  - `X-Api-Resource-Id`
  - `X-Api-Request-Id`
  - `X-Api-Sequence: -1`
- 请求体最简结构：
  - `audio.url`：音频公网 URL
  - `audio.format`：音频格式（示例写的是 mp3）
  - `request.model_name = "bigmodel"`

> 代码里已做“核心修正”：新版控制台只用 `X-Api-Key`，避免混用旧版鉴权字段。

### 2.4 查询任务与轮询（query_task + main）

- 查询函数：[asr.py:68-78](asr.py#L68-L78)
- 轮询逻辑：[asr.py:80-101](asr.py#L80-L101)

```python
def query_task(task_id, x_tt_logid):
    # 🛑 同样修正查询请求头
    headers = {
        "X-Api-Key": X_API_KEY,
        "X-Api-Resource-Id": RESOURCE_ID,
        "X-Api-Request-Id": task_id,
        "X-Tt-Logid": x_tt_logid,
        "Content-Type": "application/json"
    }
    response = requests.post(QUERY_URL, data=json.dumps({}), headers=headers, timeout=60)
    return response
```

```python
def main():
    task_id, x_tt_logid = submit_task()
    print("⏳ 识别中，自动轮询...")
    while True:
        query_response = query_task(task_id, x_tt_logid)
        code = query_response.headers.get('X-Api-Status-Code', "")
        if code == '20000000':
            result = query_response.json()
            full_text = result["result"]["text"]
            print("\n🎉 识别完成！")
            with open("识别结果.txt", "w", encoding="utf-8") as f:
                f.write(full_text)
            print("✅ 已保存到：识别结果.txt")
            exit(0)
        elif code in ('20000001', '20000002'):
            time.sleep(1)
            continue
        else:
            print(f"❌ 识别失败：{code}")
            exit(1)
```

轮询依据：响应头 `X-Api-Status-Code`
- `20000000`：识别成功，读取 `result["result"]["text"]`
- `20000001` / `20000002`：处理中（代码每 1 秒查询一次）
- 其它：识别失败，直接退出

成功后：写入 `识别结果.txt`（UTF-8 编码）。

## 5. TOS 对象存储（开通、使用方法、注意事项）

本工具的音频通过 `FILE_URL` 提供（[asr.py:8-9](asr.py#L8-L9)），该 URL 指向火山引擎 **TOS 对象存储**中的文件。因此，需要提前开通并正确配置 TOS。

### 5.1 TOS 开通与 Bucket 创建

常规流程（以控制台实际页面为准）：
1. 登录火山引擎控制台
2. 开通 **TOS（对象存储）**
3. 创建 Bucket（桶）
   - 选择地域（例如北京 `cn-beijing`）
   - 设置桶名称（示例：`audio-asr-202605`）

### 5.2 上传音频文件并获得 URL

上传方式：
- 控制台直接上传（适合少量文件）
- 使用 TOS 工具/SDK 上传（适合批量与自动化）

上传后，对象的公网 URL 通常形如：

```
https://<bucket>.<endpoint>/<object_key>
```

示例（与代码一致）：

```
https://audio-asr-202605.tos-cn-beijing.volces.com/1.mp3
```

### 5.3 权限与可访问性（非常关键）

识别服务需要从 `audio.url` 拉取音频，因此：

1. **必须确保 URL 可被公网访问**
   - 最直观判断：浏览器能否直接打开/下载该音频
2. **Bucket/Object 权限要允许读**
   - 可以将对象设置为公共读（简单但可能不安全）
   - 更安全的方式是使用“带签名的临时 URL”（如控制台/SDK 支持），并确保识别服务能访问该 URL（具体以服务能力为准）
3. **音频格式与扩展名一致**
   - 例如 mp3 文件就用 `.mp3`，并在请求中填 `format: "mp3"`

### 5.4 成本与生命周期管理建议

- TOS 通常会产生存储费与流量费（以官方计费为准）
- 建议配置生命周期规则：
  - 例如识别完成后自动删除/转存，避免长期堆积音频

## 7. 工具补充：ffmpeg（Windows 预编译版）切分大音频

下载与说明：https://ffmpeg.org/download.html#build-windows

### 7.1 简介与用途

ffmpeg 是音视频处理领域的事实标准工具。

在本项目里主要用途：
- 将**大音频文件切分为多个小片段**，降低上传与识别压力

### 7.2 安装（Windows，预编译包 + 配置 PATH）

你采用的方式是：下载预编译包 → 解压 → 把 `bin` 放到环境变量。

步骤：
1. 从上述页面进入 Windows builds，下载一个预编译发行版
2. 解压到本地目录，例如：`C:\Tools\ffmpeg\`
3. 将 `C:\Tools\ffmpeg\bin` 添加到系统环境变量 `PATH`
4. 打开新的 CMD 窗口验证：

```cmd
ffmpeg -version
```

若能输出版本信息，则安装完成。

### 7.3 一条 CMD 命令完成切分（示例）

按固定时长切分（例如每段 5 分钟 = 300 秒），尽量无损快速：

```cmd
ffmpeg -i input.mp3 -f segment -segment_time 300 -c copy output_%03d.mp3
```

说明：
- `input.mp3`：输入文件
- `-segment_time 300`：每段长度（秒）
- `-c copy`：不重新编码，速度快（但切点可能受关键帧/封装限制影响）
- `output_%03d.mp3`：输出文件名序号（000、001...）

如果发现切分点不准或 `-c copy` 不兼容，可改为重新编码（更稳但更慢）：

```cmd
ffmpeg -i input.mp3 -f segment -segment_time 300 -c:a libmp3lame -b:a 128k output_%03d.mp3
```

## 8. 常见问题排查（快速清单）

1. **提交失败**
   - 检查 `X_API_KEY` 是否为新版控制台的 X-Api-Key
2. **识别一直处理中**
   - 尝试先用 ffmpeg 切分音频
3. **识别失败 / 拉取音频失败**
   - 确认 `FILE_URL` 浏览器可直接访问
   - 检查 TOS 权限与地域 endpoint 是否正确
4. **结果文本异常**
   - 检查音频格式填写是否正确（mp3/wav）
