# 一 强化学习入门

> 强化学习资料（[磨菇书](https://datawhalechina.github.io/easy-rl/#/chapter1/chapter1?id=_111-%e5%bc%ba%e5%8c%96%e5%ad%a6%e4%b9%a0%e4%b8%8e%e7%9b%91%e7%9d%a3%e5%ad%a6%e4%b9%a0)）

**强化学习（reinforcement learning，RL）**讨论的问题是智能体（agent）怎么在复杂、不确定的环境（environment）中最大化它能获得的奖励。

如下图 1.1 所示，强化学习由两部分组成：智能体和环境。在强化学习过程中，智能体与环境一直在交互。智能体在环境中获取某个状态后，它会利用该状态输出一个动作 （action），这个动作也称为决策（decision）。然后这个动作会在环境中被执行，环境会根据智能体采取的动作，输出下一个状态以及当前这个动作带来的奖励。智能体的目的就是尽可能多地从环境中获取奖励。

![](rl-1-1.png)

强化学习和监督学习的区别如下：

1. 强化学习输入的样本是序列数据，而不像监督学习里面样本都是独立的。
2. 延迟奖励（标签）：学习器并没有告诉我们每一步正确的动作应该是什么，学习器需要自己去发现哪些动作可以带来最多的奖励，只能通过不停地尝试来发现最有利的动作。
3. 智能体获得自己能力的过程，其实是不断地试错探索（trial-and-error exploration）的过程。探索 （`exploration`）和利用（`exploitation`）**是强化学习里面非常核心的问题**。其中，探索指尝试一些新的动作， 这些新的动作有可能会使我们得到更多的奖励，也有可能使我们“一无所有”；利用指采取已知的可以获得最多奖励的动作，重复执行这个动作，因为我们知道这样做可以获得一定的奖励。因此，我们需要在**探索和利用之间进行权衡**，这也是在监督学习里面没有的情况。
4. 在强化学习过程中，没有非常强的监督者（supervisor），只有**奖励信号（reward signal**），并且奖励信号是延迟的，即环境会在很久以后告诉我们之前我们采取的动作到底是不是有效的。因为我们没有得 到即时反馈，所以智能体使用强化学习来学习就非常困难。当我们采取一个动作后，如果我们使用监督学习，我们就可以立刻获得一个指导，比如，我们现在采取了一个错误的动作，正确的动作应该是什么。而在强化学习里面，环境可能会告诉我们这个动作是错误的，但是它并没有告诉我们正确的动作是什么。而且更困难的是，它可能是在一两分钟过后告诉我们这个动作是错误的。所以这也是强化学习和监督学习不同的地方。

通过与监督学习的比较，我们可以总结出强化学习的一些特征。

- 【延迟奖励：`Delayed reward` 】强化学习会试错探索，它通过探索环境来获取对环境的理解。
- 【试错学习：`Trial and Error`】强化学习智能体会从环境里面获得延迟的奖励。
- 在强化学习的训练过程中，时间非常重要。因为我们得到的是有时间关联的数据（sequential data）， 而不是独立同分布的数据。在机器学习中，如果观测数据有非常强的关联，会使得训练非常不稳定。这也是为什么在监督学习中，我们希望数据尽量满足独立同分布，这样就可以消除数据之间的相关性。
- 智能体的动作会影响它随后得到的数据，这一点是非常重要的。在训练智能体的过程中，很多时候我们也是通过正在学习的智能体与环境交互来得到数据的。所以如果在训练过程中，智能体不能保持稳定，就会使我们采集到的数据非常糟糕。我们通过数据来训练智能体，如果数据有问题，整个训练过程就会失败。所以在强化学习里面一个非常重要的问题就是，怎么让智能体的动作一直稳定地提升。

- 【1-1】价值：将来能够获得所有奖励之和的期望值

- 【1-2】强化学习的某一个核心问题：二者的平衡

  - `exploration`：探索是否有其他更好的行动创造更大价值
  - `exploitation` ：利用已有价值函数

- 【1-3】多臂老虎机：强化学习算法的具体学习过程

  - Value：action value，一个行动具有的价值就是对应即时奖励的期望值（因为没有将来的奖励）

  - 价值函数：
    $$
    Q_t(a) = \frac{\sum_{i=1}^{t-1} R_i \cdot \mathbf{1}(A_i = a)}{\sum_{i=1}^{t-1} \mathbf{1}(A_i = a)}, \quad a = L/R
    $$
    ![](rl-1-2.png)

  - 策略函数：
    $$
    A_t = \arg\max_a Q_t(a)
    $$
    ![](rl-1-3.png)

  - `Qt(a)` 初始值不能设为0，因为一旦这样，由于贪婪策略，一开始选了一个，尝到了甜头，（而另一个没尝所以没有甜头），以后就一直会选择那一个已经尝试过的。

  - 为解决此问题，我们可以把初始值设大，鼓励更多探索（也就是让预期变的很甜，而实际上两个选择都不会这么甜，让预期的甜头变成苦头，这样他遭受毒打之后自然会选择不那么苦的另一个）

  - 但是左右老虎机策略如果改变，此方法依然失效。因此可以用 `ε-greedy`，让他有一定概率去探索另一个。

  - 具体的解释看代码即秒懂。

- 【1-5】误差：上面案例中，用到的其实就是误差学习法，适用于【**只有一个状态，并且没有延迟奖励的情况**】。

  - 公式推导：
    $$
    \begin{align*}
    \textcolor{blue}{Q_{n+1}} &= \frac{1}{n} \sum_{i=1}^{n} \textcolor{gold}{R_i} \\
    &= \frac{1}{n} \left( \textcolor{gold}{R_n} + \sum_{i=1}^{n-1} \textcolor{gold}{R_i} \right) \\
    &= \frac{1}{n} \left( \textcolor{gold}{R_n} + (n-1) \textcolor{blue}{Q_n} \right) \\
    &= \textcolor{blue}{Q_n} + \frac{1}{n} \left( \textcolor{gold}{R_n} - \textcolor{blue}{Q_n} \right)
    \end{align*}
    $$
    

    ![img](rl-1-4.png) 

  - `Qn`：对于第n次R的估计值；`Rn`：第n次的奖励；`Rn-Qn`：奖励预测误差。

  - 由于 `Qn` 存在误差，说明预测不准确，因此要将其迭代修正为 `Qn+1`，`1/n` 就是步长或者学习率。这就是**误差学习法**，也就是基于误差进行学习。

  - 但是这样还存在一个问题：学习率1/n会随着步数增加会越来越小。当环境奖励策略不变时，问题不大。但如果奖励分布随着步数增长而变化，由于我们的学习率就会越来越小，我们的预测误差就会越来越大，预测结果就会走偏。

  - 为了避免走偏，我们将学习率 `1/n` 变为常数 `α`，得到以下公式：
    $$
    \begin{align*}
    \textcolor{blue}{Q_{n+1}} &= \textcolor{blue}{Q_n} + \alpha \left( \textcolor{gold}{R_n} - \textcolor{blue}{Q_n} \right) \\
    &= (1-\alpha) \textcolor{blue}{Q_n} +  \alpha \textcolor{gold}{R_n}\\
    &= (1-\alpha)^2 \textcolor{blue}{Q_{n-1}} +  \alpha \textcolor{gold}{R_n}+\alpha(1-\alpha) \textcolor{gold}{R_{n-1}}\\
    &= (1-\alpha)^n \textcolor{blue}{Q_1} + \sum_{i=1}^{n} \alpha (1-\alpha)^{n-i} \textcolor{gold}{R_i}
    \end{align*}
    $$
    ![img](rl-1-5.png) 

  - `Qn` 不再是 `Ri` 的平均值，而变成了 `Ri` 的加权平均，而且i越大，权重越大，即更看重最近得到的奖励。这样就更加适用于**奖励分布可能会改变的情况**。

    

- 【1-6】案例：OOXX

  - 总结之前的行动价值的价值公式，也是一个基于误差的学习方法。其适用于没有延迟奖励，只有一个状态的情况。

    - 新的估计`Q(St，At)` = 旧的估计`Q(St，At)`  + 学习率`α` * 误差`Error`

    - `Q(St，At)` 代表在`St`状态下，采取`At`行动后，产生的价值`Rt`的估计。那么旧估计的误差自然就是旧估计 `Q(St，At)` - `t`时刻后所有奖励`Rt`之和。

    - 即在游戏在 `T+1` 时刻结束时，重新更新 t 时刻的行动价值函数。

    - 根据价值的定义：价值是未来所能得到的所有奖励的估计值；因此，误差为实际得到的奖励之和与旧估计之间的差值。至此，我们可以得到一个**状态行动价值**的**学习表达式**。

    - 但是这个式子在使用时，需要使用所有`Rt`的和**，**而这个Rt的和必须等到游戏运行到最终时，才会出现，所以迭代的比较慢。

      ![img](rl-1-6.png) 

  - 为了让 `t` 时刻运行之后就能迭代，我们想了个骚操作，即用估计值来代替真实值。

    - `t` 时刻后，所有奖励的值 ≈  `t` 时刻所获得的奖励 + 此后奖励的估计值（而不是真实值）
    - 与此同时，我们还可以将 `t` 至 `T` 时刻的奖励之和继续展开，将展开式带入上式。此时的学习表达式，不需要等到游戏结束就可以计算新的估计`Q(St，At)`。

  - 前者其实是蒙特卡洛方法的雏形；后者是时许差分学习法的雏形。

  - 等价地，我们可很容易写出**状态价值的学习公式**。

    ![img](rl-1-7.png) 

  - 基于以上状态行动价值函数和状态价值函数，再加上 `ε-greedy` 策略，我们就可以解决OOXX这个强化学习问题了。

    - 其实，玩家进入什么状态，不仅仅由玩家自己的行动决定，还同时受到对手行动的影响。因此，需要依据价值函数对对手的行动有一个预测。这样的方式太过复杂，目前并不采用这种方法。
    - 是否能够学习一种能够指导行动、且数量又比较少的价值？引入后果【`Outcome / Afterstate`】，这是棋牌类场景中的新概念。
    - 在玩家采取一个行动之后，直接的效应是一个后果`O`（即玩家落子之后棋牌上的棋子分布状态）。我们并不将其称为状态，因为强化学习中，状态与玩家做出的行动是一体的，玩家需要考虑在什么状态下做出什么动作。故行动的直接后果我们称为`Outcome` 而不是`State`。也就说，对手的 `Outcome` 对玩家来说是一种 `State`，玩家的 `Outcome` 对对手来说也是一种 `State`。

## 案例：多臂老虎机

我们有两个多臂老虎机（`Multi-Armed Bandit`），分别称为左边的老虎机和右边的老虎机。每个老虎机的奖励服从不同的正态分布：

- 左边的老虎机：奖励服从均值为 `500`，标准差为 `50` 的正态分布，即 `N(500,50)`。

- 右边的老虎机：奖励服从均值为 `550`，标准差为 `100` 的正态分布，即 `N(550,100)`。


我们的目标是使用 `ε-greedy` 强化学习算法【】`ε=0.1`，初始值为 `998`】来估计这两个老虎机的奖励期望值。

具体来说，我们需要通过多次尝试（拉动手臂）来逐步更新对每个老虎机奖励的估计，最终找到两个老虎机的奖励期望值。

**问题分解：**

1. 目标：
   1. 使用 ε-greedy 算法估计两个老虎机的奖励期望值。
   2. 通过多次尝试，逐步更新对每个老虎机奖励的估计。
2. ε-greedy 算法：
   1. ε=0.1：表示有 10% 的概率进行随机探索（随机选择一个老虎机），90% 的概率进行利用（选择当前估计奖励最高的老虎机）。
   2. 初始值=998：表示每个老虎机的初始奖励估计值为 998。
3. 奖励分布：
   1. 左边的老虎机：`N(500,50)`
   2. 右边的老虎机：`N(550,100)`
4. 输出：
   1. 经过多次尝试后，输出两个老虎机的奖励期望值的估计结果
   2. 通过运行代码，我们可以得到一个图表，显示两个老虎机奖励期望估计值随着时间的变化情况。随着拉动次数的增加，两个估计值应该逐渐接近它们各自的真实奖励期望值（500 和 550）

```python
import numpy as np
import matplotlib.pyplot as plt

def multi_armed_Bandit():
    # 参数初始化
    epsilon = 0.1  # ε-greedy算法中的ε
    Q1 = 998  # 左边老虎机的奖励期望估计
    Q2 = 998  # 右边老虎机的奖励期望估计
    n1 = 0  # 左边老虎机的拉动次数
    n2 = 0  # 右边老虎机的拉动次数
    num_plays = 10000  # 总共拉动的次数

    # 奖励的真实分布
    mu1, sigma1 = 500, 50  # 左边老虎机的真实奖励分布（均值，标准差）
    mu2, sigma2 = 550, 100  # 右边老虎机的真实奖励分布（均值，标准差）

    # 用于存储结果
    Q1_estimates = []
    Q2_estimates = []

    # ε-greedy策略的实验
    for t in range(num_plays):
        # 根据ε-greedy策略选择一个老虎机
        if np.random.random() < epsilon:
            action = np.random.choice([1, 2])  # 随机选择左或右
        else:
            action = 1 if Q1 > Q2 else 2  # 选择当前估计奖励最大的老虎机

        if action == 1:
            reward = np.random.normal(mu1, sigma1)  # 从左边老虎机获得奖励
            n1 += 1
            # 策略一：直接取平均值（效果不好，一直波动）
            # Q1 = (reward + Q1) / 2
            # 策略二：更新左边老虎机的奖励期望估计
            Q1 += (reward - Q1) / n1
            Q1_estimates.append(Q1)
        else:
            reward = np.random.normal(mu2, sigma2)  # 从右边老虎机获得奖励
            n2 += 1
            Q2 += (reward - Q2) / n2  # 更新右边老虎机的奖励期望估计
            Q2_estimates.append(Q2)

    # 最终的奖励期望估计
    print(f"最终左边老虎机的奖励期望估计: {Q1}")
    print(f"最终右边老虎机的奖励期望估计: {Q2}")

    # 绘图
    plt.figure(figsize=(12, 6))

    # 绘制左边老虎机奖励期望估计的变化
    plt.plot(Q1_estimates, label="Left Slot Machine (Q1)", color="blue")

    # 绘制右边老虎机奖励期望估计的变化
    plt.plot(Q2_estimates, label="Right Slot Machine (Q2)", color="red")

    # 绘制真实奖励期望值的参考线
    plt.axhline(y=mu1, color="blue", linestyle="--", label="True Q1 (500)")
    plt.axhline(y=mu2, color="red", linestyle="--", label="True Q2 (550)")

    # 图表设置
    plt.title("Reward Estimation in ε-greedy Slot Machine Experiment")
    plt.xlabel("Number of Plays")
    plt.ylabel("Estimated Reward")
    plt.legend(loc="best")
    plt.grid(True)

    # 显示图表
    plt.show()
```

## 案例：OOXX

首先定义井字棋的状态表示法：

- OOXX的棋盘一共有 9 个位置，每个位置有 3 种情况（O、X、无），对应(2、1、0)
- 所以我们用一个 9 维的向量来表示状态，每个维度表示一个位置上的 3 种情况
- 想像一下我们用一个 361 维向量来表示围棋的状态，这个向量会非常巨大
- 在实际上不可行，所以对于状态空间非常大的情况我们需要别的表示状态的方法，例如使用深度神经网络 （[具体可以参考我讲解 AlphaGo 的视频](https://www.bilibili.com/video/BV1hb4y197he) ）

![](rl-1-9.png)

对于使用强化学习来训练井字棋 `Agent`，最好的方法是训练 `Outcome` 【后果】的价值，其等价于状态行动对的价值。

- 训练好后果的价值后，我们只需要在一个状态下，选择后果价值最大的行动。
- 训练过程是使用 `New_Outcome` 来更新 `Old_Outcome`。

```python
import numpy as np
import matplotlib.pyplot as plt


# 因为我们之后需要创建两个 Agent 互相下棋，所以定义一个 Agent 的类会方便一点
class Agent:

    # 初始化，不复杂，参数只有仨
    def __init__(self, OOXX_Index, Epsilon, Alpha):
        self.index = OOXX_Index  # OOXX_Index 用 1 或者 2 代表是两个 Agent 当中的哪一个
        self.epsilon = Epsilon  # Epsilon 就是 ε-Greedy 策略中的随机选择概率
        self.alpha = Alpha  # Alpha 就是学习率

        # 储存状态价值的表
        # 注意：np.zeros((3)) 生成的是[0,0,0], np.zeros((3, 3))生成 [[0,0,0],[0,0,0],[0,0,0]]
        # 也就是说，状态相当于一个索引，价值[状态] = 某个具体的价值
        self.value = np.zeros((3, 3, 3, 3, 3, 3, 3, 3, 3))
    
        # 其实状态向量（一个状态）是唯一的，故可以作为价值矩阵的索引
        # Agent 内部记录的后果（也是一种状态），初始化为 0，表示空棋盘
        # 🔺上一个状态（旧状态，此处为旧后果）（智能体上一步行动后的棋盘状态）
        self.stored_Outcome = np.zeros(9).astype(np.int8)
        # 因为我们要将后果作为价值矩阵的索引，所以用 .astype(np.int8) 规定为整数
        # 我们要训练的也是这个后果的价值。

    # 重置后果价值
    def reset(self):
        self.stored_Outcome = np.zeros(9).astype(np.int8)

    # 输入为状态，输出为后果，同时进行价值更新
    def move(self, State):
        Outcome = State.copy()  # 拷贝一份状态
        # 因为这个状态 State 对另一名玩家来说是后果
        # 需要留着用来学习价值，所以不能直接更改，因此 .copy() 拷贝一份

        # 先判断棋盘上有哪些地方可以落子，也就是 Outcome==0 的地方
        # np.where 返回的是元组（一维）
        available = np.where(Outcome == 0)[0]

        if np.random.binomial(1, self.epsilon):  # 判断要不要采取 ε-Greedy 的随机行动【伯努利分布】
            # 随机选择一个位置标注为 1 或着 2 （取决于是 Agent1 还是 Agent2）
            Outcome[np.random.choice(available)] = self.index
        else:  # 如果不随机，就采用最优策略
            temp_Value = np.zeros(len(available))  # 创建一个临时的价值向量
            for j in range(len(available)):  # 对每一个可能落子的地方
                temp_Outcome = Outcome.copy()  # 拷贝当前时刻的状态
                temp_Outcome[available[j]] = self.index  # 假设在一个地方落子，得到后果
                temp_Value[j] = self.value[tuple(temp_Outcome)]  # 调用价值函数，计算得到的后果的价值
            choose = np.argmax(temp_Value)  # 选择价值最大的那一个行动
            # 把选择的那个位置标注为 1 或着 2 （取决于是 Agent1 还是 Agent2）
            Outcome[available[choose]] = self.index

        # 基于误差的学习法，或者说就是时序差分法 （在这个例子中即时奖励为 0）
        # 当前 Outcome的价值 - 上一轮的状态（后果）价值
        # 如果新状态比旧状态更有价值（Error为正），说明从旧状态到新状态的行动是‘好的’，应该提高旧状态的价值；反之（Error 为负），则应该降低旧状态的价值。
        # 即使用后一步的价值来修正上一步后果的价值。
        Error = self.value[tuple(Outcome)] - self.value[tuple(self.stored_Outcome)]
        # 计算当前的后果的价值估计和储存的（上一个）后果的价值估计的误差
        self.value[tuple(self.stored_Outcome)] += self.alpha * Error  # 更新储存的（上一个）后果的价值估计
        self.stored_Outcome = Outcome.copy()  # 把当前的后果储存（因为接着就要进行下一步了）
        return Outcome  # 返回当前的后果


def Judge(Outcome, curPlayer: Agent):
    """
    判断游戏是否结束
    :param Outcome:状态
    :param curPlayer:对应的玩家
    """
    Triple = np.repeat(curPlayer.index, 3)
    winner = 0  # 默认胜负未分
    if 0 not in Outcome:  # 没地方下了
        winner = 3  # 平局
    if (Outcome[0:3] == Triple).all() or (Outcome[3:6] == Triple).all() or (
            Outcome[6:9] == Triple).all():  # 分别判断三行（分别为第一行、第二行、第三行）
        winner = curPlayer.index
    # Outcome[0:7:3] 为切片语法 [start:end:step]
    # == Triple：判断是否与当前玩家的三连子匹配
    # 布尔数组.all() 是 NumPy 的方法，只有当数组中 ** 所有元素都是True** 时，才返回 True，否则返回 False。
    if (Outcome[0:7:3] == Triple).all() or (Outcome[1:8:3] == Triple).all() or (
            Outcome[2:9:3] == Triple).all():  # 分别判断三列
        winner = curPlayer.index
    if (Outcome[0:9:4] == Triple).all() or (Outcome[2:7:2] == Triple).all():  # 分别判断两条对角线
        winner = curPlayer.index
    return winner  # 返回玩家是否胜利


def Rate(Winner, Trial: int, duration: int, step: int):
    count = int((Trial - duration) / step) + 1
    Rate1 = np.zeros(count)  # Agent1 胜率
    Rate2 = np.zeros(count)  # Agent2 胜率
    Rate3 = np.zeros(count)  # 平局概率
    for i in range(count):
        Rate1[i] = np.sum(Winner[step * i:duration + step * i] == 1) / duration
        Rate2[i] = np.sum(Winner[step * i:duration + step * i] == 2) / duration
        Rate3[i] = np.sum(Winner[step * i:duration + step * i] == 3) / duration
    return Rate1, Rate2, Rate3


def OOXX_Entry():
    # 创建两个 Agent

        if i == 20000:  # 在 2 万次之后取消掉随机性
            Agent1.epsilon = 0

        winner = 0  # 默认胜负未分
        State = np.zeros(9).astype(np.int8)  # 初始化棋盘

        # Pro版

        while winner == 0:  # 如果胜负未分

                opponentPlayer.value[tuple(State)] = -1  # 对手Agent 对应的后果，也就是当前 Agent 面临的 State 的价值对 对方 Agent来说为 -1

            State = Outcome  # 更新下一轮棋盘状态
        Winner[i] = winner

    # 根据结果计算胜率
    step = 250  # 每隔250局游戏计算一次胜率
```

## 1 强化学习数学描述

**马尔可夫决策过程（`Markov Decision Process`）**：在一个具体的时刻 `t` ，智能体面临的环境处于状态 `St`；此时，智能体做出行动 `At`，得到奖励 `Rt`，同时智能体进入下状态 `S_t+1`。如此循环往复，便得到有状态、行动、奖励组成的序列。

> 为简单起见，通常假设不同的状态下，行动空间相同。

在强化学习中，我们常常认定我们所研究的决策过程具有**马尔可夫性**，即智能体获得的奖励 `Rt` 以及下一时刻状态 `S_t+1`，完全取决于当前时刻的状态 `St` 以及当前时刻的行动 `At`。也就是说，在确定了 `St` 和 `At` 的情况下，智能体获得的奖励和下一时刻状态与过去的状态和行动无关。

> 一般实际问题中也不会强调这一性质，只需要将所有需要的信息都定义在状态`S`中即可。

我们常常规定状态空间、行动空间、奖励空间都是有限的，即得到 **有限马尔可夫决策过程（`Finite Markov Decision Process`）**。

另外，我们考虑的问题一般为离散的，即强化学习的研究主要**基于离散的有限马尔可夫决策过程**。

> 对于连续问题，也可以进行离散化；同时，强化学习中的很多原理和方法都可以推广到连续问题上。

![](rl-2-1.png)

在离散的有限马尔可夫决策过程中，还具有一定的随机性。比如智能体在一个特定状态下选择了一个特定的行动，得到的奖励和下一时刻状态都是随机的。

![](rl-2-2.png)

注意：此处的有限指的是状态空间、行动空间、奖励空间是有限的；离散是时间上的离散。而根据时间是否有限，强化学习可以划分为两种强化学习问题：

- 存在终止状态 `S_T` 的强化学习问题，马尔可夫决策过程会在终止状态终止，并且重新开始下一次的决策过程。这种问题被称为 `Episodic`。
- 不存在终止状态的强化学习问题，被称为 `Continuing`。

<br>

接下来，我们开始对目标、策略、价值进行数学上的定义。

### 1.1 目标 `Goal`

强化学习中将目标定义为未来所获得的奖励之和（回报 `Return`），而强化学习的目标也是将目标最大化。根据是否存在终止状态，可分别定义目标`Goal`。

> 对于无限连续强化学习问题，需要引入折扣因子`γ`。这也符合常识，即同样价值下更看重短期利益（立马获得￥100与一年后获得￥100）。

![](rl-2-3.png)

<br>

### 1.2 策略 `Policy`

策略是一个函数，输入是状态，输出是行动。但一般情况下，智能体所选择的行动并不是确定的。

因此，我们使用一个策略函数 `Π` 来表示策略。它表示在状态 `S` 的情况下，智能体选择行动 `a` 的概率。

![](rl-2-4.png)

<br>

### 1.3 价值 `Value`

价值，是将来所能获得奖励之和的期望值，因此我们可以通过回报 `Return` 来定义价值。

- 状态价值：处于状态 `S` 时，将来所能获得奖励之和的期望值。
- 状态行动价值：处于状态 `S`，且采取行动 `a` 时，将来所能获得奖励之和的期望值。

> 通常，当智能体策略改变时，状态-行动价值也会变化，故价值往往是基于策略的。

![](rl-2-5.png)

## 2 动态规划：`Dynamic Programming`

### 2.1 动态规划引入

动态规划（`Dynamic Programming`）：强化学习基础方法之一，将复杂问题分解为简单的子问题。

> 一个例子：从`x`出发到`z`，如何选择路径，才能获得最多的奖励呢？

![](rl-2-6.png)

- 对于情形一，各条路径上的奖励用 `A/B/C/D` 表示，如何获得最大奖励的路径组合方案呢？
  - 首先，这属于一个强化学习问题，即 `x/y/z` 为三个状态，`A/B/C/D` 为行动。
  - 最直接的方法：暴力枚举。其实总共也就4种组合方案，问题在于情形二种的情况。
  - 子问题的思路：x到y是一个子问题，y到z是一个子问题，每个子问题都选择奖励最大的路径，即可分而治之。
  - 但如果对于更复杂的情形三，仅仅子问题的思路又不够解决。

- 对于情形二，假如有N个状态，每个状态有M个行动，又将如何？
  - 此时，暴力枚举的总数为 `M` 的 `N-1` 次方，当`M`、`N`很大时，将无法承受计算量。

- 对于情形三，显然只知道A和B两个选择的即时奖励是不够的，我们还需要知道后续从m和n出发能获得奖励之和。
  - 引入新的定义：**状态最优价值**。一个状态的状态最优价值，就是指在**最优策略下**的**状态价值**，即某个点处的未来能够获得的最大奖励之和。
  - 使用 `v*` 表示状态最优价，`*`表示最优策略。显然，`m`点处的状态最优价值为2，`n`点处的状态最优价值为4。
  - 在状态最优价值的引导下，`x` 点处需要将即使奖励与m、n点的状态最优价值相加来确定如何选择。


<br>

> 情形三中的子问题，其实代表了所有确定的强化学习问题的子问题，因此从情形三开始推广。

智能体从状态 `s` 开始，可选行动 `a`【确定的行动a】，奖励为 `r`【确定的即时奖励r】，到达状态 `s'`。

- 要解决这个子问题，就需要对每一个行动计算出：即时奖励 + 下一个状态 `s'` 的状态最优价值。
- 最优策略就应该是：选择使这个和值最大的行动。
- 状态 `s` 下能够获得的所有奖励之和的最大值，或者说状态 `s` 在最优策略下的状态价值，即就是**状态 `s` 的状态最优价值**。
- 状态 `s` 的状态最优价值方程，是强化学习的核心方程之一，即**贝尔曼最优方程**（Bellman Optimality Equation）。
- 贝尔曼最优方程中，方括号内的内容是：状态`s`下选择了行动`a`之后，能过获得的最大所有奖励之和。我们将其称之为**状态-行动最优价值**，记为 `q*(s,a)`。

因此，我们认识到，**状态最优价值**就是解决强化学习问题的关键所在。如何计算状态最优价值呢？

- 首先，在不知道状态最优价值的前提下，可以对其进行估值，通常初值为0，使用`^`冒号表示估计值。
- 之后，进入贝尔曼公式的迭代流程，即使用 `s'` 的状态最优价值的估计值来更新 `s` 的状态最优价值的估计值。
- 同时，也可以对未来价值添加折扣因子`γ`。
- 只要我们确保，对每个状态的估计值都不断进行迭代，那么这些估计值最终就会收敛到真实的状态最优价值。这也是我们研究**有限**马尔可夫决策过程的原因之一，即只有当状态空间有限的情况下，我们才能保证价值估计的收敛。

> 注意：再次强调，此处的讨论和定义针对的强化学习问题，均是确定的、有限的！

- 对于随机的、离散的强化学习问题，可以由贝尔曼公式推广。
- 对于确定的、连续的最优控制问题，贝尔曼公式会推广为 **哈密顿-雅可比-贝尔曼方程**。

### 2.2 案例：迷宫探险【确定版】

> 本案例是针对确定的、离散的强化学习问题。

问题介绍：3x3的迷宫路径规划，目标为获得最大的奖励并走出迷宫。规则和建模设计如下：

- 实线表示无法通行，虚线表示可通行。
- 迷宫的两个出口位于做左下角和右下角，左下角出口能够得到+10%额外奖励并走出迷宫，右下角出口会被木乃伊吃掉。
  - 左下角和右下角的出口定义为两个终止状态，其状态最优价值为0。
- 迷宫中的玩家每次可以选择最多四种行动，即上下左右。
- 玩家每次移动都会消耗食物和水，即算作 `-1` 的即时奖励。
- 玩家可以选择向墙面运动，但只会停留在原位，并且同样得到 `-1` 的即时奖励。

![graph](rl-2-8.png)

强化学习相关的建模设计：

- 初始化每个位置状态最优价值，分为两个确定的实际值（金色）与多个估计值。
- 每个状态保留一个旧估计值和一个新估计值（蓝色）。
- 为简单起见，设置折扣率`γ=1`。

<br>

**算法实现思路：**

![graph](rl-2-9.png)

1. **状态价值迭代：**每个状态最优价值不断迭代，直到不再随迭代发生变化，就可得到准群的最优价值。
   - 对于新旧估计迭代策略，我们采用策略二，即等待每一个状态都计算出新估计值之后，再统一用新估计值替换掉旧的估计值。
   - 策略一：如果我们得到一个状态的新的估计值之后，就立马就用来估计其他状态。对于这种策略：
     - 这样的算法通常收敛的更快，并且具体的收敛过程会取决于我们对不同状态进行迭代的顺序。
     - 通常，我们实际上可以采用任意顺序进行迭代，甚至随机顺序迭代，我们只需要保证对每一个状态都不断进行迭代即可。
   - 策略二：反之，我们也可以等每一个状态都计算出新估计值之后，再统一用新估计值替换掉旧的估计值。
     - 这样的算法不会受到迭代顺序的影响，收敛速度慢。
     - 这里，为简单统一起见，我们采用此策略。
2. **状态行动价值迭代：**与上类似，最终计算出准确的状态-行动最优价值。

![graph](rl-2-10.png)

<br>

- 新增武器和额外奖励，有武器的情况下可以由右下角出口离开迷宫，并且获得+18%的额外奖励。

```python
"""

#         0       1       2
#     ╔═══════╦═══════╦═══════╗
# 0   ║   +4  |       ║ Sword ║
#     ╠───────────────║───────╣
# 1   ║       |       |       ║
#     ╠════════───────════════╣
# 2 → |   T   |       |   T   | ←
#     ╚═══════╩═══════╩═══════╝
#         ↑               ↑
#        +10         +18 (Sword)
import numpy as np


class Maze:
    def __init__(self):
        self.state_dimension = tuple((3, 3, 2, 2))  # 状态的维度为 3×3×2×2 的矩阵
        # 前两个维度代表位置，第三个维度代表左上角的额外奖励是否还存在，第四个维度代表右上角的武器是否还存在
        self.terminate = np.array([[2, 0], [2, 2]])  # 左下角和右下角为终止状态
        self.all_actions = np.array([[-1, 0], [1, 0], [0, -1], [0, 1]])  # 上下左右四个行动
        self.wall = np.array([[[0, 1], [0, 2]], [[1, 0], [2, 0]], [[1, 2], [2, 2]]])  # 墙的位置

    # 判断是否为不可能的状态
    def judge_impossible(self, state):
        position = state[0:2]
        if (position == np.array([0, 0])).all() and state[2] == 1:  # 玩家到达左上角并且额外奖励还存在是不可能的
            return 1
        if (position == np.array([0, 2])).all() and state[3] == 1:  # 玩家到达右上角并且武器还存在是不可能的
            return 1
        return 0

    # 判断是否到达终止状态
    def judge_terminate(self, position):
        is_terminate = 1 in np.all(position == self.terminate, axis=1)
        return is_terminate

    # 判断是否穿墙
    def judge_wall(self, current_position, next_position):
        is_wall_1 = 1 in np.all(np.array([current_position, next_position]) == self.wall, axis=(1, 2))
        is_wall_2 = 1 in np.all(np.array([next_position, current_position]) == self.wall, axis=(1, 2))
        return is_wall_1 or is_wall_2

    # 状态转移，输入当前状态和行动，输出下一时刻的状态和即时奖励
    def state_transition(self, current_state, action):
        current_position = current_state[0:2]  # 读取位置信息
        next_position = np.clip(current_position + action, 0, 2)  # np.clip() 确保不会超出迷宫的边缘
        if self.judge_wall(current_position, next_position):  # 如果撞墙
            next_position = current_position.copy()  # 状态不变
        next_state = current_state.copy()  # 下一时刻的状态
        next_state[0:2] = next_position  # 首先更新位置信息
        # 然后更新其他信息，同时计算奖励
        bonus = 0  # 额外奖励
        if (next_position == np.array([2, 0])).all():  # 左下角终止状态额外奖励为 10
            bonus = 10
        if (next_position == np.array([0, 0])).all() and current_state[2] == 1:  # 左上角额外奖励为 4，前提是它存在
            next_state[2] = 0  # 现在不存在了
            bonus = 4
        if (next_position == np.array([0, 2])).all() and current_state[3] == 1:  # 右上角获得武器，前提是它存在
            next_state[3] = 0  # 现在不存在了
        if (next_position == np.array([2, 2])).all() and current_state[3] == 0:  # 右下角击杀木乃伊获得 18 额外奖励，前提是有武器
            bonus = 18
        reward = -1 + bonus
        return reward, next_state


class Agent:
    def __init__(self, Gamma: float = 1, Precision=1e-10):
        self.state_optimal_value = None
        self.gamma = Gamma  # 折扣率
        self.precision = Precision  # 判断是否收敛的精度

    # 价值迭代的主要函数
    def value_iteration(self, maze: Maze):
        dimension = maze.state_dimension  # 环境给出的状态的维度，在这里是 3×3×2×2 的矩阵 dimension = (3,3,2,2)
        old_value = np.zeros(dimension)  # np.zeros((3,3,2,2)) 储存旧价值估计
        new_value = np.zeros(dimension)  # np.zeros((3,3,2,2)) 储存新价值估计
        Row = dimension[0]  # 行数 = 3
        Column = dimension[1]  # 列数 = 3
        delta = 1  # 初始默认新旧价值还差得远
        while delta > self.precision:
            for i in range(Row):  # 遍历每一行
                for j in range(Column):  # 遍历每一列
                    for m in range(2):  # 左上角额外奖励是否存在
                        for n in range(2):  # 右上角武器是否存在
                            current_state = np.array([i, j, m, n])  # 当前状态
                            current_position = current_state[0:2]  # 读取位置信息
                            # 判断是否是终止状态或者不可能的状态
                            if not (maze.judge_terminate(current_position) or maze.judge_impossible(current_state)):
                                # 临时的价值向量，储存状态-行动最优价值的估计值
                                temp_value = np.zeros(len(maze.all_actions))
                                for k in range(len(maze.all_actions)):  # 遍历所有可能的行动
                                    # 调用环境的状态转移函数，得到即时奖励和下一时刻的状态
                                    reward, next_state = maze.state_transition(current_state, maze.all_actions[k])
                                    # 状态-行动最优价值的估计值 = 即时奖励 + γ×下一时刻的状态最优价值的估计值
                                    temp_value[k] = reward + self.gamma * old_value[tuple(next_state.astype(int))]
                                # 新的状态最优价值的估计值 = 最大的状态-行动最优价值的估计值
                                new_value[i, j, m, n] = np.max(temp_value)
            delta = np.max(np.abs(new_value - old_value))  # 计算新旧估计的差值，取绝对值，再取最大值
            old_value = new_value.copy()  # 用新的估计值代替旧的估计值
        self.state_optimal_value = new_value.copy()  # 收敛之后把状态最优价值储存

    # 最优策略函数
    def optimal_policy(self, Maze, state):
        # 和价值迭代的过程一样，只是现在我们利用价值迭代得到的状态最优价值
        temp_value = np.zeros(len(Maze.all_actions))
        for k in range(len(Maze.all_actions)):
            reward, next_state = Maze.state_transition(state, Maze.all_actions[k])
            temp_value[k] = reward + self.gamma * self.state_optimal_value[tuple(next_state.astype(int))]
        # 选择状态-行动最优价值最大的行动
        optimal_k = np.argmax(temp_value)
        optimal_action = Maze.all_actions[optimal_k]
        return optimal_action

    # 给一个初始状态，返回最优路径，以及回报值
    def optimal_trajectory(self, Maze, initial_state):
        total_return = 0  # 记录回报值
        power = 0  # 折扣率的指数，每一步都增加 1
        all_state = np.array([initial_state])  # 记录路径中的所有状态
        current_state = initial_state.copy()
        terminate = 0
        while not terminate:
            optimal_action = self.optimal_policy(Maze, current_state)
            reward, next_state = Maze.state_transition(current_state, optimal_action)
            total_return += reward * self.gamma ** power  # 将即时奖励添加到回报值当中
            power += 1  # 下一时刻的奖励将会打折扣
            all_state = np.r_[all_state, np.array([next_state])]  # 把下一时刻的状态添加到路径当中
            current_state = next_state.copy()  # 新状态替换旧状态
            terminate = Maze.judge_terminate(current_state[0:2])  # 判断是否到达终止状态
        return all_state, total_return


def maze_entry():
    maze = Maze()  # 创建一个迷宫环境
    agent = Agent(Gamma=0.9)  # 创建一个玩家
    agent.value_iteration(maze)  # 价值迭代
    print("状态最优价值：")
    print(agent.state_optimal_value[:, :, 1, 1])
    print("最优路径：")
    all_state, total_return = agent.optimal_trajectory(maze, np.array([0, 1, 1, 1]))  # 从第一行中间位置出发
    print(all_state)
    print("奖励之和：")
    print(total_return)
```

## 3 随机动态规划

### 3.1 状态随机转移概率

引入**状态转移的随机概率**：`p(r, s'| s, a)` ，其表示在状态`s`、行动为`a`的情况下，得到的奖励为`r`，并且下一时刻的状态为 `s'` 的概率。

![graph](rl-2-12.png)

即可得到 **随机的贝尔曼最优方程**，唯一区别就是增加了取期望的过程。

<br>

> 一个游戏案例：硬币正反对赌。

- 左右玩家有不同数量的筹码，左侧玩家为Boss，由右侧玩家上交筹码。
- 硬币为正面则左侧玩家赢得相应的筹码，否则Boss输掉相应的筹码。
- 游戏结束条件为其中一方玩家的筹码数量为0。
- 假设全场只有4个筹码，那么此问题只有五种状态，其中两种终止状态的最优价值为0。
- 假设硬币为正面的概率为0.6。

强化学习建模：

![graph](rl-2-13.png)

- 当玩家（右侧）只有1个筹码，那么将有`0.6`的概率输掉所有（进入状态0），有`0.4`的概率赢得一个（进入状态2）;
- 当玩家有3个筹码，那么Boss手中只有1个筹码，玩家最多放入一个筹码。因此，玩家将有`0.4`的概率赢得所有（进入状态4，获得 `r=1` 的即时奖励，其他状态转移的即时奖励为0），有`0.6`的概率输掉一个（进入状态3）;
- 此时，我们可以计算状态1与状态3的状态最优价值。
- 当玩家由两个筹码，则两种情况：全部放入，或者只放一个。针对两种情况，分别计算状态最优价值并取最优策略。

### 3.2 案例：迷宫探险【随机的价值迭代版】

相对于确定版迷宫，随机版迷宫新增规则如下：

- 玩家每走一步，都有`0.1`的概率踩到陷阱，并且获得`-10`的额外即时奖励。



![graph](rl-2-14.png)

```python
#         0       1       2
#     ╔═══════╦═══════╦═══════╗
# 0   ║   +4  |       ║ Sword ║
#     ╠───────────────║───────╣
# 1   ║       |       |       ║
#     ╠════════───────════════╣
# 2 → |   T   |       |   T   | ←
#     ╚═══════╩═══════╩═══════╝
#         ↑               ↑
#        +10         +18 (Sword)
import numpy as np
# 加入状态转移的随机概率，即随机的、离散的强化学习问题
# 1、新增 state_transition_stochastic()
# 2、修改 Agent.value_iteration()  optimal_policy()

class Maze():
    def __init__(self):
        self.state_dimension = tuple((3, 3, 2, 2))  # 状态的维度为 3×3×2×2 的矩阵
        # 前两个维度代表位置，第三个维度代表左上角的额外奖励是否还存在，第四个维度代表右上角的武器是否还存在
        self.terminate = np.array([[2, 0], [2, 2]])  # 左下角和右下角为终止状态
        self.all_actions = np.array([[-1, 0], [1, 0], [0, -1], [0, 1]])  # 上下左右四个行动
        self.wall = np.array([[[0, 1], [0, 2]], [[1, 0], [2, 0]], [[1, 2], [2, 2]]])  # 墙的位置

    # 判断是否为不可能的状态
    def judge_impossible(self, state):
        position = state[0:2]
        if (position == np.array([0, 0])).all() and state[2] == 1:  # 玩家到达左上角并且额外奖励还存在是不可能的
            return 1
        if (position == np.array([0, 2])).all() and state[3] == 1:  # 玩家到达右上角并且武器还存在是不可能的
            return 1
        return 0

    # 判断是否到达终止状态
    def judge_terminate(self, position):
        is_terminate = 1 in np.all(position == self.terminate, axis=1)
        return is_terminate

    # 判断是否穿墙
    def judge_wall(self, current_position, next_position):
        is_wall_1 = 1 in np.all(np.array([current_position, next_position]) == self.wall, axis=(1, 2))
        is_wall_2 = 1 in np.all(np.array([next_position, current_position]) == self.wall, axis=(1, 2))
        return is_wall_1 or is_wall_2

    # 状态转移，输入当前状态和行动，输出下一时刻的状态和即时奖励
    def state_transition(self, current_state, action):
        current_position = current_state[0:2]  # 读取位置信息
        next_position = np.clip(current_position + action, 0, 2)  # np.clip() 确保不会超出迷宫的边缘
        if self.judge_wall(current_position, next_position):  # 如果撞墙
            next_position = current_position.copy()  # 状态不变
        next_state = current_state.copy()  # 下一时刻的状态
        next_state[0:2] = next_position  # 首先更新位置信息
        # 然后更新其他信息，同时计算奖励
        bonus = 0  # 额外奖励
        if (next_position == np.array([2, 0])).all():  # 左下角终止状态额外奖励为 10
            bonus = 10
        if (next_position == np.array([0, 0])).all() and current_state[2] == 1:  # 左上角额外奖励为 4，前提是它存在
            next_state[2] = 0  # 现在不存在了
            bonus = 4
        if (next_position == np.array([0, 2])).all() and current_state[3] == 1:  # 右上角获得武器，前提是它存在
            next_state[3] = 0  # 现在不存在了
        if (next_position == np.array([2, 2])).all() and current_state[3] == 0:  # 右下角击杀木乃伊获得 18 额外奖励，前提是有武器
            bonus = 18
        reward = -1 + bonus
        return reward, next_state

    # 随机状态转移，输入当前状态和行动，输出所有可能得下一时刻的状态和即时奖励，以及对应的概率
    def state_transition_stochastic(self, current_state, action):
        reward, next_state = self.state_transition(current_state, action)
        all_probability = np.array([0.9, 0.1])  # 0.1的概率踩到陷阱
        all_reward = np.array([reward, reward - 10])  # 踩到陷阱得到-10的额外即时奖励
        all_next_state = np.array([next_state, next_state])
        return all_reward, all_next_state, all_probability

                    for m in range(2):  # 左上角额外奖励是否存在
                        for n in range(2):  # 右上角武器是否存在
```

### 3.3 一般的贝尔曼方程

根据前面的内容，我们掌握了**贝尔曼最优方程**以及**最优价值迭代算法**。

> 我们使用`*`代表最优策略，在最优策略下，我们选择状态-行动价值最大的行动，即方程中的`max`。

下面我们将**贝尔曼最优方程推广**到**一般的贝尔曼方程**。

![graph](rl-2-15.png)

我们使用一个概率分布 `Π` 来表示在一个状态 `s` 下选择行动 `a` 的概率。

要得到一般的策略对应的贝尔曼方程，我们只需要将贝尔曼最优方程中的”最优“去掉，即去掉`max`，并替换为对概率分布`Π`的期望值；同时将代表最优策略的`*`替换为代表一般策略的`Π`。

> 策略评估和（最优）价值迭代唯一的区别：前者取期望，后者取最大值。

- 价值迭代算法可以直接收敛到状态最优价值，从而得到最优策略。

- 策略评估只会收敛到一般策略对应的状态价值，因此还需要再得到一个策略对应的状态价值后，对这个策略进行改进。改进方法如下：

  1. 给定任意一个策略`Π`，对其进行策略评估，直到收敛，从而得到这个策略下的状态价值。
  2. 在策略`Π`的基础上进行改进，得到新的策略`Π'`。我们让策略`Π'`去选择策略`Π`对应的状态-行动价值最大的行动（若存在多个行动的状态-行动价值都是最大的，那就平均分配一个概率），这样的得到`Π'`一定比原本的`Π`要更优，并不一定是最优。
  3. 我们在对策略`Π'`进行策略评估，得到策略`Π'`对应的状态价值，然后在对`Π'`进行改进，得到一个更新的策略`Π''`。
  4. 循环往复，直到收敛到最优策略，状态价值也会随之收敛到状态最优价值。

  ![graph](rl-2-16.png)

> 如果策略已经收敛，即新旧策略不再发生改变那么策略改进公式就可以用同一个`Π'`表示。此时计算`Π'`的状态价值，即贝尔曼最优方程，即收敛到最优策略。

**引申出的一个强化学习原理：**

在策略迭代当中，我们一边对策略进行评估，目的是得到这个策略下的状态价值；另一边是对策略进行改进，目的是让新的策略去选择状态-行动价值最大的行动。

我们其实并不需要不断进行”价值迭代“，使得价值估计收敛到真实值；我们只需要让新的价值估计比旧的价值估计更准确一些就够了。实际上，我们只进行一次迭代，那么这个算法和（最优）价值迭代是完全等价的。

我们其实也不需要让新的策略一定去选择状态-行动价值最大的行动；我们只需要让新的策略比旧的策略更优一些就够了，例如稍提高一些选择状态-行动价值最大的行动的概率。

总的来说，我们实际上只需要**对当前的策略，得到一个更准确的价值估计**，然后**根据当前的价值估计，得到一个更好的策略**。不断循环，策略会收敛到最优策略，价值估计会收敛到最优价值估计。

绝大多数强化学习算法，都建立在这个原理之上，如蒙特卡洛法和时序差分法等等。

# 三 深度强化学习DQN

深度Q网络（Deep Q-Network, DQN）是深度强化学习的里程碑算法，它将传统Q学习与深度神经网络结合，能够处理高维状态空间。本文通过一个简化的“单停机位航班排序”问题，详细拆解DQN的训练思路和步骤。

## 1 问题设定：单停机位航班排序

为了聚焦DQN的训练过程，我们将之前复杂的多停机位问题简化为**只有一个停机位**，有3个航班需要按时间顺序安排，目标是最大化使用廊桥的航班数（奖励+1）。

每个航班有到达时间、离开时间和是否可停靠廊桥的属性。停机位可停靠任意航班，但必须满足时间间隔约束。

> 状态空间很小，便于手工计算演示。

### 环境要素

- **航班数据**（简化）：
    
    |航班ID|到达时间|离开时间|可廊桥？|
    |-----|-------|-------|-------|
    |--0--|---0---|---5---|---是--|
    |--1--|---3---|---8---|---否--|
    |--2--|---6---|---10--|---是--|
    
- **安全间隔**：假设前机离开后需至少间隔1分钟才能安排后机。
- **动作**：从当前未分配的航班中选择一个作为下一个分配。
- **状态**：当前已分配的航班序列（或最后分配的航班ID）以及当前时间（即最后航班的离开时间+间隔）。为简化，我们用“最后航班ID”和“当前时间”表示状态，但为了神经网络输入，我们会编码。
- **奖励**：若所选航班可停靠廊桥（属性为是），则奖励+1，否则0。
- **终止**：所有航班分配完毕。

## 2 DQN核心思想回顾

DQN的目标是学习一个最优动作价值函数 Q∗(s,a)，即在状态 s 下采取动作 a 后，遵循最优策略所能获得的期望累计折扣奖励。Q学习使用贝尔曼方程迭代更新：



Q(s,a)←Q(s,a)+α[r+γmax⁡a′Q(s′,a′)−Q(s,a)]

DQN用神经网络 Q(s,a;θ)Q(s,a;θ) 近似Q函数，并通过梯度下降最小化损失：

L(θ)=E(s,a,r,s′)∼D[(r+γmax⁡a′Q(s′,a′;θ−)−Q(s,a;θ))2]L(θ)=E(s,a,r,s′)∼D​[(r+γa′max​Q(s′,a′;θ−)−Q(s,a;θ))2]

其中：

- DD 是经验回放池，存储历史转移。
    
- θ−θ− 是目标网络的参数，定期从 θθ 复制，用于稳定训练。
    
- γγ 是折扣因子。
    

## 3 训练准备

对于我们的单停机位问题，状态可以定义为：

- 最后航班的离开时间（或当前可用时间）

输入层4个神经元，隐藏层若干（如8个神经元），输出层3个神经元（对应3个航班动作）。激活函数ReLU。

## 5 关键设计原理解释

#### 1. 为什么需要经验回放？

- **打破样本相关性**：连续交互产生的样本高度相关，直接顺序学习会导致网络振荡或过拟合。随机采样回放池中的历史样本，使样本独立同分布，提高训练稳定性。
    
- **提高样本效率**：每个转移可被多次使用，避免样本浪费。
    

#### 2. 为什么需要目标网络？

- 在TD更新中，目标值 r+γmax⁡a′Q(s′,a′)r+γmaxa′​Q(s′,a′) 依赖于当前Q网络的输出，若两者同时更新，容易导致训练不稳定（追逐一个移动的目标）。使用单独的目标网络，其参数固定一段时间，提供稳定的目标值，使Q网络更新更平稳。
    

#### 3. ϵϵ-greedy探索的作用

- 初期ϵϵ较大，鼓励智能体随机探索未知动作，积累多样化的经验。随着训练进行，ϵϵ逐渐衰减，智能体更多利用学到的Q值选择最优动作，实现探索与利用的平衡。
    

#### 4. 折扣因子 γγ 的意义

- 控制未来奖励的重要性。γγ 接近1表示长远规划，接近0表示短视。在航班调度中，我们希望最大化全程的廊桥使用数，因此需要一定前瞻性。
    

## 6 与多停机位问题的联系

上述单停机位示例虽然简单，但训练框架完全适用于多停机位问题，区别仅在于：

- **状态空间更大**：需要包含所有停机位的最后航班ID（通过嵌入层处理）和全局分配掩码。
    
- **动作空间更大**：每个（停机位，航班）对作为一个动作，需用掩码屏蔽非法动作。
    
- **网络结构**：输入维度增加，输出维度变为 M×N，但损失计算和更新方式完全相同。
    
- **合法动作处理**：在目标值计算中，max⁡a′Q(s′,a′)maxa′​Q(s′,a′) 仅考虑合法动作，否则会选出非法动作导致错误。因此，必须在计算目标时也使用掩码。
    

## 7 DQN训练流程总结（伪代码）

```python
初始化 Q 网络 θ，目标网络 θ⁻ = θ
初始化经验回放池 D
for episode = 1 to M:
    初始化环境，得到状态 s
    while not done:
        根据 ε-greedy 选择动作 a（用 θ 预测 Q 值，并应用合法掩码）
        执行 a，得到奖励 r 和下一状态 s'
        存储 (s, a, r, s') 到 D
        if 终止: done = True
        if D 中样本足够:
            从 D 随机采样一个批次 B
            对每个样本计算目标值 y:
                if s' 终止: y = r
                else: y = r + γ * max_{a'} Q_target(s', a')  # 应用合法掩码
            计算损失 L = MSE(y, Q(s, a; θ))
            梯度下降更新 θ
        每隔 C 步: θ⁻ = θ
        s = s'
    ε 衰减
```

### 八、结语

通过这个简化的航班排序例子，你应该能清楚看到DQN如何利用经验回放和目标网络从交互数据中学习最优策略。虽然实际应用中的状态和动作空间可能非常庞大，但核心训练逻辑是一致的。理解这些基础后，再回头看你之前设计的多停机位DQN，就能明白为什么需要嵌入层处理`last_flight`，以及如何通过掩码确保动作合法性。训练的本质就是不断最小化贝尔曼方程左右两侧的差异，让Q网络逼近真实的Q函数。
