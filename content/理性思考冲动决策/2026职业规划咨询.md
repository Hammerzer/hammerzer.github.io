# 一 简历问题

## 1.简历内容问题

1. 教育背景顺序有问题，应弱化民航大学相关表述，可按本科先写民航大学，再写硕士相关内容或者调整顺序，把个人评价或专业技能放最上面，工作实习分开写。
2. 简历中工作实习部分问题多，如汉朔的工作写前后端不合适，应体现专精内容；白云实习方向较专，但未体现算法应用等相关亮点。
3. 投递方向乱，C++和Python都能投很多方向，但没有针对投递方向优化简历，对目标公司业务和技术方向缺乏了解。

## 1.2 项目相关问题

1. 简历上项目体现不出方向，如研究生运筹学方向与实习项目没有关联，简历显得杂乱无章，应突出与目标方向相关的项目，例如如果投游戏方向，应体现游戏相关的项目经验。
2. 对于C++项目，目前只有基础和进阶学习情况，没有完整项目体现，如Qt项目没写上去或者未完成未写，影响简历质量。


# 二 职业选择与规划

## 2.1 适合的职业方向

1. 工程开发+算法方向比较适合，如智能调度系统开发（结合物流、航空等领域）、高性能系统开发等。
2. 不建议优先考虑纯大模型算法岗，因为要求高（学历、算法能力等方面），而其本身岗位较少且多集中于头部公司。
3. 传统算法如优化推荐、NLP方向目前招聘收缩且与自身背景不太适配。
4. C++方向可选择游戏、音视频、高性能系统等方向；如果补充语言，Go可作为补充用于分布式或云相关方向，但不是必须，Java也可作为补充，尤其在分布式或算法应用方面。

## 2.2 职业规划建议

1. 先根据自身背景确定几个可能的方向，如物流航空调度、智慧城市、医疗信息化、金融风控等，结合业务场景进行规划。
2. 在面试过程中不断调整简历方向，按行业梯度面试，从练手公司开始逐步向理想公司迈进。
3. 对于职业发展，可设定2 - 3年有一次职位或业务变动的时间节点，避免到30 - 35岁时还处于专员岗位而无晋升。
4. 一定要结合业务提升自己的专业能力，一定要深耕！

# 三 焦虑

## 3.1 焦虑来源

1. 就业反馈不好，投递简历无回应，担心选错技术方向。
2. 看到AI等技术发展迅速，担心自己被替代，如AI辅助编码让程序员有无力感。
3. 担心未来的职业稳定性，在考虑是否选择稳定但可能发展空间有限的方向（如嵌入式软硬件结合方向因自身基础薄弱不适合）。

## 3.2 焦虑的建议

1. 先解决眼前的就业问题，调整简历、确定方向去投递获取反馈，而不是先纠结长远的技术发展方向。
2. 要认识到AI虽然发展迅速但不会完全替代程序员，程序员的核心在于**应用场景的理解和利用AI完成特定任务的能力**，而不是单纯担心被替代。
3. 长期是长期，找工作是找工作。
4. "一人公司"虽然已经存在，但毕竟只是轻资产的项目，对于重资产的工业（如机场航空）短期内无法做到AI化。
5. 一定要做！

# 四 其他补充

1. 关于项目实践
    1. 计划做Qt项目、C++网络库项目，还有涉及OpenGL（用于工业仿真软件）的项目，部分项目计划一两周完成，应先将已完成或即将完成的项目写进简历投递。
    2. 在项目中要体现业务逻辑而非纯技术，例如做加密播放器项目（Qt相关且有音视频内容）应强调其工业终端应用场景以便投递相关方向公司。
2. 长期发展思考
    - 虽然要学习AI相关知识（算法刷题等），但当下还是要以C++为主线，将AI与C++结合，在工程能力基础上探索AI应用。
    - 对于新兴方向如自动化驾驶（要求较高，涉及操作系统、强化学习、目标识别、嵌入式等多方面知识）可以关注但不强求，要根据自身情况选择合适的发展方向。
