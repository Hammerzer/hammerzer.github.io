# Obsidian 使用技巧

>[!TIP] 关于 Obsidian 的官方文档，[传送门](https://publish.obsidian.md/help-zh/%E7%94%B1%E6%AD%A4%E5%BC%80%E5%A7%8B)在此！

## 一 🎯 基础操作技巧

### 1.1 快速导航
- **打开快速切换器**：`Ctrl/Cmd + O` - 快速打开笔记、搜索标题
- **跳转到链接**：`Ctrl/Cmd + 点击链接` - 直接跳转到笔记
- **返回上一页**：`Alt + 左箭头` - 浏览器式导航

### 1.2 编辑快捷键
- **加粗**：`Ctrl/Cmd + B`
- **斜体**：`Ctrl/Cmd + I`
- **删除线**：`Alt + Shift + 5`
- **插入链接**：`Ctrl/Cmd + K`
- **插入图片**：`Ctrl/Cmd + Alt + I`
- **插入代码块**：`Ctrl/Cmd + Alt + C`

### 1.3 笔记管理
- **新建笔记**：`Ctrl/Cmd + N` - 在当前目录新建笔记
- **新建文件夹**：右键文件夹 → 新建文件夹
- **重命名**：`F2` 或右键 → 重命名
- **删除**：`Delete` 键或右键 → 删除

## 二 ✍️ 高效写作技巧

### 2.1 双向链接
```markdown
[[笔记名称]]          # 基本双向链接
[[笔记名称|别名]]     # 带别名的双向链接
[[#标题]]             # 页面内锚点链接
[[笔记名称#标题]]     # 跨笔记标题链接
```

### 2.2 标签系统
```markdown
#标签名称            # 基本标签
#分类/子分类          # 层级标签
```

**使用技巧**：
- 使用层级标签（如 `#前端/JavaScript`）组织内容
- 在搜索栏输入 `tag:#标签名` 搜索该标签的所有笔记
- 在侧边栏"标签"面板管理和浏览标签

### 2.3 模板功能
**创建模板**：
1. 在 `content/` 目录下创建 `templates/` 文件夹
2. 创建模板文件（如 `笔记模板.md`）
3. 在设置中配置模板文件夹路径
4. 使用 `Ctrl/Cmd + Alt + T` 插入模板

**示例模板**：
```markdown
---
title: {{title}}
description:
date: {{date:YYYY-MM-DD}}
tags: []
---

# {{title}}

> 摘要/简介

## 内容

## 参考资料
```

### 2.4 块引用和链接
```markdown
# 标题 ^header1       # 给标题添加块引用

文本内容 ^block1      # 给文本块添加引用

[[笔记名称#^header1]] # 链接到标题块
[[笔记名称#^block1]]  # 链接到文本块
```

### 2.7 私密笔记保护

在 Obsidian 管理个人知识库并通过 Quartz 发布为数字花园时，保护私密笔记是核心需求。我为你整理了**从简单到复杂、从阻止发布到内容加密**的完整方案，确保你能精细控制哪些内容公开、哪些完全私密。

| 方案              | 核心原理                 | 安全级别  | 适用场景             |
| --------------- | -------------------- | ----- | ---------------- |
| draft/ignore 标记 | Quartz 构建时直接排除，不生成页面 | ★★★☆☆ | 不想发布但无需加密的笔记     |
| 全局目录过滤          | 配置文件批量排除指定文件夹        | ★★★☆☆ | 大量私密内容按目录管理      |
| explorerexclude | 从导航 / 搜索隐藏但 URL 仍可访问 | ★★☆☆☆ | 临时隐藏、需手动分享的内容    |
| Obsidian 加密插件   | 内容级加密，即使误发布也无法阅读     | ★★★★★ | 高度敏感内容（密码、财务、日记） |
| 混合方案            | 标记 + 加密 + 过滤三重保障     | ★★★★★ | 核心机密、个人隐私等高风险内容  |
#### Quartz 层面 - 阻止私密笔记被发布（推荐）

这是最基础、最常用的私密保护方式，直接在 Quartz 构建过程中排除私密内容，**完全不会生成对应的网页文件**。

在私密笔记（单篇笔记标记）的 YAML frontmatter 中添加 `draft: true`：
```markdown
---
title: "我的私密日记"
draft: true  # 关键配置：Quartz 将不生成此页面
---

这是我的个人日记，仅本地可见，不会出现在网站上
```

**操作步骤**：

1. 打开需要保密的笔记
2. 在顶部添加 `---` 包围的 YAML 区域
3. 加入 `draft: true`
4. 重启 Quartz 服务：`npx quartz dev`（确保 ContentIndexing 插件已启用）

## 三 🎨 工作区优化

### 3.1 侧边栏管理
- **文件面板**：点击文件夹名称旁的箭头展开/折叠
- **搜索面板**：搜索笔记内容和标题
- **标签面板**：按标签浏览笔记
- **大纲面板**：显示当前笔记的标题层级
- **反向链接面板**：显示引用当前笔记的其他笔记

### 3.2 分屏功能
- **垂直分屏**：右键标签 → 垂直分屏
- **水平分屏**：右键标签 → 水平分屏
- **移动标签**：拖动标签到分屏区域

### 3.3 视图模式
- **阅读模式**：`Ctrl/Cmd + E` - 专注阅读
- **编辑模式**：`Ctrl/Cmd + E` - 编辑内容
- **源模式**：`Ctrl/Cmd + Shift + E` - 查看原始 Markdown 源码

## 四 🔍 搜索与导航

### 4.1 高级搜索
- **搜索笔记内容**：在搜索栏输入关键词
- **搜索标题**：`intitle:关键词`
- **搜索标签**：`tag:关键词`
- **搜索路径**：`path:关键词`
- **组合搜索**：`intitle:前端 tag:JavaScript`

### 4.2 图视图
- **打开图视图**：点击左侧边栏的图图标
- **图视图布局**：力导向图、分层布局
- **筛选节点**：搜索特定笔记或标签
- **调整大小**：滚轮缩放，拖拽节点

### 4.3 快速切换
- **切换笔记**：`Ctrl/Cmd + O` 快速搜索并切换
- **最近打开**：点击左侧边栏"打开的文件"
- **历史记录**：`Ctrl/Cmd + Shift + O` 查看历史记录

## 六 📚 知识管理技巧

### 6.1 笔记命名规范
```
[类型]_[主题].md           # 如：教程_JavaScript基础.md
[日期]_[主题].md          # 如：20260208_ReactHooks.md
[项目]_[功能].md          # 如：电商_购物车组件.md
```

### 6.2 文件夹结构
```
content/
├── 编程语言/              # JavaScript, Python, Java 等
├── 前端开发/              # HTML, CSS, React, Vue 等
├── 后端开发/              # Node.js, 数据库, 架构等
├── 工具与资源/            # Git, VSCode, 插件等
├── 学习笔记/              # 读书、课程笔记等
├── 项目经验/              # 实战项目总结
└── 杂项/                  # 生活笔记、灵感等
```

### 6.3 笔记整理流程
1. **收集**：快速记录想法，不追求完美
2. **整理**：定期整理笔记，添加标签和链接
3. **链接**：建立笔记间的关联（双向链接）
4. **回顾**：定期复习重要笔记
5. **删除/归档**：处理过时内容

### 6.4 知识体系构建
- **主题分类**：按主题建立文件夹
- **核心概念**：创建核心概念笔记并链接其他相关笔记
- **学习路径**：创建学习路线图笔记，链接相关资源
- **项目文档**：为每个项目创建专门的笔记文件夹

## 八 🔧 常见问题解决

### 8.1 笔记链接失效
**问题**：笔记移动位置后，链接失效
**解决**：使用"检查链接"功能，修复断链

### 8.2 搜索结果不完整
**问题**：搜索时找不到某些笔记
**解决**：重建搜索索引（设置 → 搜索 → 重建索引）

### 8.3 性能问题
**问题**：笔记数量多时，Obsidian 变慢
**解决**：
- 关闭不必要的插件
- 优化图片大小
- 定期清理缓存
